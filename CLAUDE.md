## Project Overview

This is a classic Snake game built with TypeScript, HTML5 Canvas, and tsup. The game features:
- Modular architecture with separation of concerns
- Canvas-based rendering with smooth animations
- Keyboard controls and responsive UI
- Score tracking with localStorage persistence
- Configurable game settings (speed, grid size)

## Common Commands

All commands should be run from the `tanchishe/` directory:

```bash
cd tanchishe
npm install          # Install dependencies
npm run dev          # Development mode with hot reload
npm run build        # Production build (minified)
npm run preview      # Preview production build
npm run clean        # Remove dist directory
```

## Architecture

The game follows a modular, component-based architecture:

### Core Components
1. **Game** (`src/game.ts`) - Central coordinator
   - Manages game state (score, running status, etc.)
   - Controls game loop using `requestAnimationFrame`
   - Handles game lifecycle (start, pause, reset)

2. **Snake** (`src/snake.ts`) - Player-controlled entity
   - Maintains body segments as array of `Position` objects
   - Handles movement, direction changes, and growth
   - Implements collision detection (walls, self)

3. **Food** (`src/food.ts`) - Collectible items
   - Random generation avoiding snake positions
   - Collision detection with snake head

4. **GameRenderer** (`src/gameRenderer.ts`) - Canvas rendering
   - Draws grid, snake, food, and UI elements
   - Separates rendering logic from game state
   - Updates DOM elements for scores and stats

5. **InputHandler** (`src/inputHandler.ts`) - User input
   - Keyboard controls (arrow keys, space, R)
   - UI button event handlers
   - Prevents opposite direction changes

6. **Type Definitions** (`src/types.ts`) - Type safety
   - Centralized interfaces for `GameState`, `Position`, `Direction`
   - Ensures type consistency across components

### Game State Management
The `GameState` interface (`types.ts`) centralizes all game data:
- Snake and food objects
- Score, high score, and statistics
- Game status (running, paused, game over)
- Configuration (grid size, colors, FPS)

### Rendering Pipeline
1. Game loop updates state based on input and collisions
2. `GameRenderer.render()` draws current state to canvas
3. UI elements updated separately via DOM manipulation

- Source maps generated

1. `npm run dev` starts development server with watch mode
2. Changes to TypeScript files trigger automatic rebuild
3. Built files output to `dist/` with source maps
4. HTML and CSS served from `public/` directory

## Key Implementation Details

### Game Loop
- Uses `requestAnimationFrame` with FPS control
- Update rate configurable via `GAME_CONFIG.fps`
- Separates update logic from rendering

### Collision Detection
- Wall collision: Checks if head is outside grid bounds
- Self collision: Compares head position with body segments
- Food collision: Checks head position against food position

### State Persistence
- High score stored in `localStorage`
- Game configuration constants in `GAME_CONFIG` object
- No server-side dependencies

### UI Integration
- Canvas element for game rendering (`#game-canvas`)
- Overlay system for start/pause/game over screens
- Real-time score and stat updates
- Responsive design with CSS Grid/Flexbox
